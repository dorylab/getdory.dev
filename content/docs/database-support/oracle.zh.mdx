# Oracle

<img src="/icons/databases/oracle.svg" alt="Oracle logo" width="48" height="48" />

Dory 支持连接 Oracle Database，适合企业系统、ERP 数据、财务流程、报表平台和长期运行的生产数据库。连接后，你可以浏览 Schema 和数据库对象，运行 Oracle SQL，预览数据，并让 AI 基于 Oracle 上下文辅助生成、解释或修复查询。

## 适合的 Oracle 使用场景

- 查询 ERP、财务、审计、客户和运营数据。
- 浏览 Schema、表、视图、索引、主键、函数和存储过程。
- 用 AI 生成 Oracle 报表 SQL 或排查 SQL。
- 避免在 Oracle 查询中误用 `LIMIT` 等其他数据库方言。
- 保存常用 Oracle 查询，方便后续复用。

## 支持能力

| 能力 | Oracle 支持情况 |
| --- | --- |
| Connections | 支持 Host、Port、Service Name、Username、Password、SSL 和 SSH Tunnel。 |
| Explorer | 支持浏览 Schema、表、视图、字段、索引、主键、函数和存储过程。 |
| SQL Console | 支持 Oracle SQL 编写、运行和结果查看。 |
| AI Chat | 可基于 Schema 生成 Oracle SQL、解释查询和修复错误。 |
| Charts & Results | 支持对查询结果做可视化分析。 |
| Saved Queries | 支持保存常用 Oracle 查询。 |

## 查询建议

Oracle 的限制行数和分页通常使用 `FETCH FIRST n ROWS ONLY` 或 `OFFSET ... FETCH NEXT ...`。如果让 AI 编写 Oracle SQL，请明确目标数据库是 Oracle，这样它会考虑 `dual`、标识符大小写、命名参数和 Oracle catalog views 等差异。

## 连接前需要准备

- Host、Port、Service Name、Username 和 Password。
- Dory 运行环境到 Oracle 的网络访问。
- 如环境要求，准备 SSL 或 SSH Tunnel 配置。
- 当前账号需要具备读取 Schema 元数据和目标对象的权限。
