# Dory Docs

Dory is an AI-native SQL client for humans and agents. Humans can query, explore, and visualize data, while AI agents can safely work with databases through MCP and leave their work in the same editable SQL workspace.

## One Workspace for Humans and Agents

Humans and agents use the same underlying database actions:

- **Humans** write SQL, browse schemas, inspect result tables, create charts, and save useful queries.
- **Agents** list connections, explore schemas, run read-only SQL, and organize analysis through MCP.
- **The workspace** preserves SQL tabs, result snapshots, charts, saved queries, and execution context so a person can inspect, edit, rerun, and continue the work.

## Dory vs. a Plain Database MCP Server

| Capability | Plain database MCP server | Dory |
| --- | ---: | ---: |
| Run SQL and inspect schema | Yes | Yes |
| Editable SQL tabs and persistent results | No | Yes |
| Charts, filters, and saved queries | Limited | Yes |
| Human review and continuation | Limited | Yes |
| Works as a daily SQL client | No | Yes |
