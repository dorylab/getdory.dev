# Dory Documentation

Dory is an AI-native data workspace for modern databases. It combines a schema-aware SQL console, conversational AI, database exploration, saved queries, and charting into one workspace.

## What this site covers

- How to get Dory running through the demo, desktop app, Homebrew, or Docker.
- How to connect supported databases and move from schema discovery to your first useful query.
- How Dory's core features support SQL editing, AI workflows, exploration, saved queries, and charts.
- What is available today for deployment, references, and release tracking.

## Recommended path

1. Start with [Introduction](./getting-started).
2. Follow [Connect Your Database](./getting-started/connect-your-database).
3. Learn the [SQL Console](./core-features/sql-console) and [AI Chat](./core-features/ai-chat).
4. Review [Deploy](./deploy) before rolling Dory out to a team.

## Current product scope

Based on the public repository, Dory currently documents support for:

- ClickHouse with deep operational integration.
- PostgreSQL, Neon, MySQL, MariaDB, SQLite, and DuckDB.
- Pluggable AI providers including OpenAI, Anthropic, Google, Qwen, xAI, and OpenAI-compatible APIs.

For recent product changes, see [Release Notes](./release-notes).
