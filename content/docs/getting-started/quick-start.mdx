# Quick Start

Choose the shortest path to the outcome you want.

## Try the Live SQL Workspace

Open the [Dory live demo](https://app.getdory.dev), select **Sign in as demo**, and use the included data to explore schema, run SQL, inspect results, and create a chart. No installation is required.

## Start a Local SQL Workspace

Install Dory Desktop:

```bash
brew install dorylab/dory/dory
```

You can also download macOS or Windows builds from [GitHub Releases](https://github.com/dorylab/dory/releases). Then:

1. Open Dory.
2. Create a connection with read-only credentials.
3. Continue with [Connect Your Database](/en/docs/getting-started/connect-your-database).

## Connect a Local Agent

If Dory Desktop is installed, open **Settings → Agent Access**, enable MCP, and add the displayed endpoint to your client. The default endpoint is:

```text
http://127.0.0.1:3318/api/mcp
```

For Codex CLI:

```bash
codex mcp add dory --url http://127.0.0.1:3318/api/mcp
codex mcp list
```

Dory manages the Desktop grant, so normal Desktop users do not need to copy a bearer token. For standalone, headless, or hosted setups, use [MCP Setup](/en/docs/agents/mcp-setup).

## Run Dory Web

Use the Web Docker image when a team needs browser access. Start with the production-aware [Docker guide](/en/docs/deploy/docker), which covers required secrets, persistence, and the public URL.

## Expected Outcome

You should finish with either a working SQL workspace or a connected MCP client. Continue with [First Query in 3 Minutes](/en/docs/getting-started/first-query-in-3-minutes) or [Agent Runs](/en/docs/agents/agent-runs).
