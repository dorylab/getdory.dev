# はじめに

Dory は対話型 SQL ワークスペースとしても、エージェントのデータベース実行レイヤーとしても利用できます。目的に合う実行形態を選んでください。

## 実行形態を選ぶ

| 実行形態 | 適した用途 | 次のページ |
| --- | --- | --- |
| Dory Desktop | 日常の SQL 作業と Desktop 接続を再利用するローカル エージェント | [Dory のインストール](/ja/docs/getting-started/installation)、[データベース接続](/ja/docs/getting-started/connect-your-database)。 |
| Dory Web | ブラウザー利用、共有チーム、セルフホスト | [クイックスタート](/ja/docs/getting-started/quick-start)または[デプロイ](/ja/docs/deploy)。 |
| Dory CLI | Desktop を開かないローカル エージェント、CI、自動化 | [MCP セットアップ](/ja/docs/agents/mcp-setup)と [CLI & Headless Runtime](/ja/docs/agents/cli-headless)。 |
| Headless Runtime | 安定した HTTP MCP エンドポイントとエージェント専用サーバー | [CLI & Headless Runtime](/ja/docs/agents/cli-headless)。 |

## 最初の人間向けワークフロー

1. [Dory をインストールまたは起動](/ja/docs/getting-started/quick-start)します。
2. [データベースを接続](/ja/docs/getting-started/connect-your-database)します。
3. [3 分で最初のクエリ](/ja/docs/getting-started/first-query-in-3-minutes)を実行します。
4. 結果を確認し、チャートまたは保存済みクエリとして残します。

## 最初のエージェント ワークフロー

1. [MCP セットアップ](/ja/docs/agents/mcp-setup)で Desktop MCP、standalone stdio、headless HTTP、hosted bridge から選びます。
2. エージェントに Schema の確認と読み取り専用 SQL の実行を依頼します。
3. Dory で Agent Run を開き、保存された SQL タブと結果スナップショットから作業を続けます。

本番データには読み取り専用のデータベース資格情報を使用し、権限を拡大する前に生成 SQL を確認してください。
