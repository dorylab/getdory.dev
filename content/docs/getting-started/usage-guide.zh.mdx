# Dory 使用文档

Dory 是一个面向 SQL 与 AI 的数据工作台。你可以在同一个工作区里管理数据库连接、浏览 Schema、编写和运行 SQL、让 AI 生成或优化查询，并把有价值的查询保存下来复用。

这篇文档适合作为第一次使用 Dory 的完整路线图。

## 1. 进入 Dory

你可以通过以下方式进入 Dory：

- 在线演示：打开 `https://app.getdory.dev`，选择 **Enter as Demo** 或以访客身份继续。
- 桌面端：在 macOS 上通过 Homebrew 安装，或从 GitHub Releases 下载桌面安装包。
- 自托管：使用 Docker 在自己的服务器或本地环境运行 Dory。

如果只是体验功能，建议先使用在线演示或桌面端。团队长期使用时，建议阅读 [自托管](../deploy/self-hosting) 与 [Docker](../deploy/docker)。

## 2. 创建或选择工作区

登录后，Dory 会进入当前组织或访客工作区。工作区用于隔离数据库连接、保存查询、成员设置和 MCP 访问令牌。

常见使用方式：

- 个人试用：直接使用默认工作区或访客工作区。
- 团队共用：创建组织，统一管理成员、连接和权限。
- 演示环境：使用 Demo 入口加载示例数据，快速体验 SQL Console 和 AI 能力。

## 3. 添加数据库连接

打开 **Connections**，点击 **Add Connection**，按表单填写连接信息。

Dory 当前支持的连接类型包括：

| 类型 | 适用场景 |
| --- | --- |
| ClickHouse | 日志、事件、可观测性和高吞吐分析场景。 |
| PostgreSQL / Neon | 业务系统、产品数据、通用分析。 |
| MySQL / MariaDB | Web 应用数据库和运营报表。 |
| SQLite | 本地文件、轻量演示和可复现实验。 |
| DuckDB / MotherDuck | 本地分析文件、嵌入式分析和云端 DuckDB 工作流。 |

连接表单支持常规 Host / Port / Database / Username / Password，也支持按数据库类型填写文件路径、连接串、SSL 和 SSH Tunnel。保存前建议先点击 **Test Connection**。

## 4. 浏览数据库结构

连接成功后，进入连接工作区。建议先打开 **Explorer** 或左侧 Schema 面板，确认：

- 数据库和 Schema 是否能正常加载。
- 表、视图和字段是否完整显示。
- 字段类型、主键、分区键、排序键、备注等信息是否符合预期。
- 是否可以打开表详情页查看结构、DDL、数据预览和统计信息。

面对陌生数据库时，先浏览结构再写 SQL，可以显著减少字段猜测和错误查询。

## 5. 在 SQL Console 编写和运行查询

打开 **SQL Console** 后，你可以使用多标签页并行分析。推荐流程：

1. 选择当前数据库或 Schema。
2. 从左侧表列表插入 `SELECT` 起始查询，或直接手写 SQL。
3. 先运行带 `limit` 的小查询验证数据。
4. 查看结果表格、行数、耗时和错误信息。
5. 根据结果继续修改 SQL，直到输出结构稳定。

SQL Console 会结合当前连接提供表名、字段名和语法提示。Dory 默认会限制返回行数，适合先做探索，再逐步扩大查询范围。

## 6. 使用 AI 辅助分析

Dory 的 AI 能读取当前连接、Schema、SQL 和查询结果上下文。你可以在 SQL Console 的 Copilot 面板或 AI Chat 中使用它。

常见用法：

- **生成 SQL**：用自然语言描述目标，例如“统计最近 7 天每小时订单数”。
- **修复 SQL**：把报错 SQL 交给 AI，结合错误信息生成修正版。
- **优化 SQL**：让 AI 重写复杂查询、拆分 CTE 或减少扫描范围。
- **解释 SQL**：理解 JOIN、窗口函数、聚合口径和字段含义。
- **继续追问结果**：基于当前结果继续拆分维度、找异常或生成下一步查询。

生成的 SQL 建议先人工审核，再在目标数据库执行。生产库推荐使用只读账号。

## 7. 查看结果与生成图表

查询完成后，先在结果表格确认数据是否正确，再进入图表或分析能力。

适合可视化的数据通常具有清晰结构：

- 趋势图：一列时间 + 一列或多列指标。
- 柱状图：一列分类维度 + 一列指标。
- 饼图：少量分类 + 占比指标。
- 明细表：用于排查问题或导出检查。

如果图表效果不理想，通常应该先调整 SQL：明确列名、提前聚合、减少无关字段，并在 SQL 中完成排序。

## 8. 保存和复用查询

当一条查询已经验证有效，可以保存到 **Saved Queries**。

建议保存：

- 需要反复运行的业务指标查询。
- 已经调优过、后续还会复用的 SQL。
- 团队成员需要参考的排查脚本。
- 图表或分析流程依赖的基础查询。

命名时尽量使用业务目标，例如“最近 30 天 DAU 趋势”，不要只写“users query”。

## 9. 使用 ClickHouse 专属能力

如果连接的是 ClickHouse，Dory 还提供更深入的操作能力：

- **Monitoring**：查看查询总量、慢查询、错误查询、P50 / P95 耗时、QPM、最近查询和 SQL 详情。
- **Privileges**：管理用户、角色、全局权限和数据库 / 表 / 视图级权限，并支持集群级操作。

这些能力依赖 ClickHouse 系统表和权限。建议使用具备必要只读监控权限或管理员权限的账号。

## 10. 接入外部 AI Agent

Dory 提供 MCP 访问能力。你可以在设置中创建个人令牌，把 Dory 连接、Schema、只读 SQL、保存查询和监控摘要暴露给支持 MCP 的客户端。

典型用途：

- 在 Codex、Claude 等工具里查询 Dory 连接列表。
- 让 Agent 搜索数据库 Schema 或读取表结构。
- 执行受限制的只读 SQL。
- 基于结果构建分析上下文或图表建议。

详细配置见 [MCP / Agent Access](../reference/mcp)。
