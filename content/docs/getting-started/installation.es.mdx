# Instalación

Dory ofrece puntos de entrada distintos para SQL interactivo, despliegues Web compartidos y runtimes solo para agentes.

## Dory Desktop

```bash
brew install dorylab/dory/dory
```

Descarga instaladores para macOS y Windows desde [GitHub Releases](https://github.com/dorylab/dory/releases). Desktop es la opción recomendada para SQL diario, archivos de base de datos locales y Desktop MCP.

## Dory Web

Usa la imagen `dorylab/dory` para una interfaz de navegador y un espacio compartido. Un despliegue real requiere secretos, almacenamiento persistente y una URL pública correcta; sigue la guía de [Docker](/es/docs/deploy/docker).

## Dory CLI

`@getdory/cli` ejecuta Dory sin abrir Desktop y requiere Node.js 20 o posterior.

```bash
npx -y @getdory/cli --help
```

Para un servidor de larga duración:

```bash
npm install -g @getdory/cli
dory --help
```

La CLI admite los modos de datos `standalone`, `desktop` y `self-hosted`. Continúa con [CLI & Headless Runtime](/es/docs/agents/cli-headless).

## Headless Docker

Usa `dorylab/dory-headless` cuando los agentes necesiten MCP por HTTP sin interfaz Web. Consulta [CLI & Headless Runtime](/es/docs/agents/cli-headless) para Token, volúmenes persistentes y TLS.

## Verificación

- Desktop abre [Connections](/es/docs/core-features/connections).
- Web tiene almacenamiento persistente y una URL de inicio de sesión correcta.
- CLI supera `dory doctor --data standalone` después de inicializar.
- Headless HTTP está protegido por Bearer Token y no se publica sin TLS.
