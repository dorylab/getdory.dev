# 开始使用

Dory 既可以作为交互式 SQL 工作台，也可以作为 Agent 的数据库执行层。先根据目标选择合适的运行形态。

## 选择运行形态

| 运行形态 | 适合场景 | 下一步 |
| --- | --- | --- |
| Dory Desktop | 日常 SQL 工作，以及复用 Desktop 连接的本地 Agent | [安装 Dory](/zh/docs/getting-started/installation)，然后[连接数据库](/zh/docs/getting-started/connect-your-database)。 |
| Dory Web | 浏览器访问、团队共享和自托管部署 | 阅读[快速开始](/zh/docs/getting-started/quick-start)或[部署](/zh/docs/deploy)。 |
| Dory CLI | 不打开桌面应用的本地 Agent、CI 和自动化 | 阅读 [MCP Setup](/zh/docs/agents/mcp-setup)和 [CLI & Headless Runtime](/zh/docs/agents/cli-headless)。 |
| Headless Runtime | 稳定的 HTTP MCP 端点和纯 Agent 服务器 | 阅读 [CLI & Headless Runtime](/zh/docs/agents/cli-headless)。 |

## 第一个人类工作流

1. [安装或打开 Dory](/zh/docs/getting-started/quick-start)。
2. [连接数据库](/zh/docs/getting-started/connect-your-database)。
3. 按照[三分钟完成第一个查询](/zh/docs/getting-started/first-query-in-3-minutes)运行受限、可审阅的查询。
4. 检查结果、创建图表或保存查询。

## 第一个 Agent 工作流

1. 在 [MCP Setup](/zh/docs/agents/mcp-setup) 中选择 Desktop MCP、standalone stdio、headless HTTP 或 hosted bridge。
2. 让 Agent 探索 Schema 并运行只读 SQL。
3. 在 Dory 中打开生成的 Agent Run，从保存的 SQL 标签页和结果快照继续工作。

连接生产数据时应使用只读数据库凭据，并在扩大权限前审阅生成的 SQL。
