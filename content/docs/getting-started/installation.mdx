# Installation

Dory has separate entry points for interactive SQL work, shared Web deployments, and agent-only runtimes.

## Dory Desktop

Install on macOS with Homebrew:

```bash
brew install dorylab/dory/dory
```

Download macOS and Windows installers from [GitHub Releases](https://github.com/dorylab/dory/releases). Desktop is the recommended choice for daily SQL work, local database files, and Desktop MCP.

## Dory Web

Use the `dorylab/dory` image for a browser UI and shared team workspace. A real deployment requires application secrets, persistent storage, and a correct public URL; follow [Docker](/en/docs/deploy/docker) rather than relying on a bare container command.

## Dory CLI

`@getdory/cli` runs Dory without opening the desktop app. It requires Node.js 20 or newer.

```bash
npx -y @getdory/cli --help
```

Install it globally for a long-running server:

```bash
npm install -g @getdory/cli
dory --help
```

The CLI supports `standalone`, `desktop`, and `self-hosted` data modes. Continue with [CLI & Headless Runtime](/en/docs/agents/cli-headless).

## Headless Docker

Use `dorylab/dory-headless` when agents or automation need MCP over HTTP without the Web UI. Token creation, persistent volumes, and TLS guidance are covered in [CLI & Headless Runtime](/en/docs/agents/cli-headless).

## Verify the Path You Chose

- Desktop opens and can reach [Connections](/en/docs/core-features/connections).
- Web has persistent application storage and a working sign-in URL.
- CLI returns help and passes `dory doctor --data standalone` after initialization.
- Headless HTTP is protected by a bearer token and is not exposed publicly without TLS.
