# Primeros pasos

Dory puede funcionar como espacio SQL interactivo o como capa de ejecución de bases de datos para un agente. Elige la modalidad adecuada para tu trabajo.

## Elige una modalidad

| Modalidad | Adecuada para | Continúa con |
| --- | --- | --- |
| Dory Desktop | SQL diario y agentes locales que reutilizan conexiones de Desktop | [Instala Dory](/es/docs/getting-started/installation) y [conecta una base de datos](/es/docs/getting-started/connect-your-database). |
| Dory Web | Navegador, equipos compartidos y despliegues autohospedados | [Inicio rápido](/es/docs/getting-started/quick-start) o [Deploy](/es/docs/deploy). |
| Dory CLI | Agentes locales, CI y automatización sin abrir Desktop | [MCP Setup](/es/docs/agents/mcp-setup) y [CLI & Headless Runtime](/es/docs/agents/cli-headless). |
| Headless Runtime | Endpoints HTTP MCP estables y servidores solo para agentes | [CLI & Headless Runtime](/es/docs/agents/cli-headless). |

## Primer flujo para una persona

1. [Instala o abre Dory](/es/docs/getting-started/quick-start).
2. [Conecta una base de datos](/es/docs/getting-started/connect-your-database).
3. Ejecuta una consulta limitada con [Primera consulta en 3 minutos](/es/docs/getting-started/first-query-in-3-minutes).
4. Revisa el resultado, crea un gráfico o guarda la consulta.

## Primer flujo para un agente

1. Elige Desktop MCP, standalone stdio, headless HTTP o hosted bridge en [MCP Setup](/es/docs/agents/mcp-setup).
2. Pide al agente que explore el esquema y ejecute SQL de solo lectura.
3. Abre el Agent Run en Dory y continúa desde las pestañas SQL y resultados guardados.

Usa credenciales de base de datos de solo lectura en producción y revisa el SQL generado antes de ampliar permisos.
