# Getting Started

Dory can run as an interactive SQL workspace or as the database execution layer behind an agent. Choose the runtime that matches the work you want to do.

## Choose a Runtime

| Runtime | Best for | Continue with |
| --- | --- | --- |
| Dory Desktop | Daily SQL work and local agents that should reuse Desktop connections | [Install Dory](/en/docs/getting-started/installation), then [connect a database](/en/docs/getting-started/connect-your-database). |
| Dory Web | Browser access, shared teams, and self-hosted deployments | [Quick Start](/en/docs/getting-started/quick-start) or [Deploy](/en/docs/deploy). |
| Dory CLI | Local agents, CI, and automation without opening the desktop app | [MCP Setup](/en/docs/agents/mcp-setup) and [CLI & Headless Runtime](/en/docs/agents/cli-headless). |
| Headless Runtime | Stable HTTP MCP endpoints and agent-only servers | [CLI & Headless Runtime](/en/docs/agents/cli-headless). |

## First Human Workflow

1. [Install or open Dory](/en/docs/getting-started/quick-start).
2. [Connect a database](/en/docs/getting-started/connect-your-database).
3. Run a limited, reviewed query with [First Query in 3 Minutes](/en/docs/getting-started/first-query-in-3-minutes).
4. Inspect the result, create a chart, or save the query.

## First Agent Workflow

1. Choose Desktop MCP, standalone stdio, headless HTTP, or a hosted bridge in [MCP Setup](/en/docs/agents/mcp-setup).
2. Ask the agent to inspect schema and run read-only SQL.
3. Open the resulting Agent Run in Dory and continue from the saved SQL tabs and result snapshots.

Use read-only database credentials for production data and review generated SQL before expanding access.
