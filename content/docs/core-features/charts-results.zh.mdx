# Charts & Results

结果面板是判断查询是否正确的第一现场。Dory 支持你先检查原始结果，再在输出稳定后继续做图表展示。

## 结果表格

运行 SQL 后，Dory 会展示查询状态、耗时、返回行数和结果表格。建议先检查：

- 列名是否表达清楚。
- 指标口径是否正确。
- 是否存在明显空值或重复行。
- 结果行数是否符合预期。
- 排序是否稳定。

## 推荐顺序

1. 先验证原始结果集。
2. 确认列类型和列名是否合理。
3. 必要时在 SQL 中补充分组或排序。
4. 选择和数据形态匹配的图表。

## 适合做图表的场景

- 时间序列趋势。
- Top N 对比。
- 分布统计。
- 分类拆分。

## 常见图表数据结构

| 目标 | 推荐结果结构 |
| --- | --- |
| 趋势 | `time_bucket` + `metric` |
| 分类对比 | `category` + `metric` |
| 占比 | `category` + `value`，分类数量不要过多 |
| 多指标趋势 | `time_bucket` + 多个指标列 |
| 明细排查 | 保留关键 ID、状态、时间和错误字段 |

## 让 AI 帮你整理结果

如果结果不适合画图，可以让 AI 帮你把 SQL 改成更适合可视化的结构。例如：

- 把当前结果改成按天聚合的趋势图 SQL。
- 只保留 Top 10 分类，其余归为 Other。
- 把多个状态字段转换成适合柱状图的输出。
- 根据当前结果推荐图表类型。

AI 生成的查询仍然需要你确认业务口径，尤其是时间范围、去重逻辑和聚合方式。
