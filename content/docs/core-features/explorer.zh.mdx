# Explorer

Explorer 让你在写 SQL 之前先快速理解数据库结构，尤其适合面对陌生 Schema 时使用。

## 可以查看什么

- 可用的数据库和 Schema。
- 表、视图、物化视图和函数等对象。
- 字段名和字段类型。
- 字段备注、主键、分区键、排序键、默认值和可空性。
- 表 DDL、属性、索引、统计信息和数据预览。
- 不同对象之间的关系或命名模式。

## 为什么重要

Explorer 可以显著减少猜测，也能提升 AI 提示质量，因为你可以引用真实对象，而不是用模糊描述。

## 推荐使用流程

1. 选择当前连接。
2. 在 Explorer 中选择数据库和 Schema。
3. 从表名、视图名和字段名判断核心业务对象。
4. 打开表详情，先看 **Overview** 和 **Structure**。
5. 使用 **Data Preview** 查看少量样本。
6. 复制或引用真实字段名，再进入 SQL Console 编写查询。

## 表详情页

表详情通常包含以下标签：

| 标签 | 用途 |
| --- | --- |
| Overview | 查看摘要、关键字段、查询片段和语义信息。 |
| Data Preview | 快速预览少量数据。 |
| Structure | 查看字段、约束、DDL、属性和索引。 |
| Stats | 查看数据库提供的表统计信息。 |

不同数据库能提供的元数据不完全一致。如果某个标签为空，通常是当前数据库驱动或账号权限无法读取对应信息。

## 与 AI 配合

当你不确定某张表是否适合使用时，可以先在 Explorer 中确认字段，再向 AI Chat 提问：

- 这张表更像明细表还是聚合表？
- 哪些字段适合作为时间维度？
- 基于这些字段生成一条趋势查询。
- 解释这些字段可能代表的业务含义。
