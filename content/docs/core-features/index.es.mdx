# SQL Workspace

Dory sigue siendo un cliente SQL rápido y familiar. El mismo espacio recibe SQL y resultados creados por agentes para que una persona pueda revisar y continuar sin reconstruir el chat.

## Superficies SQL diarias

| Función | Uso |
| --- | --- |
| [Connections](/es/docs/core-features/connections) | Credenciales, red y archivos locales. |
| [Explorer](/es/docs/core-features/explorer) | Bases, esquemas, tablas, columnas, previews y objetos. |
| [SQL Console](/es/docs/core-features/sql-console) | Escribir, ejecutar, depurar y guardar SQL en varias pestañas. |
| [Charts & Results](/es/docs/core-features/charts-results) | Revisar filas reales, filtrar y crear gráficos. |
| [AI Chat](/es/docs/core-features/ai-chat) | Generar, explicar, corregir o adaptar SQL dentro de Dory. |
| [Saved Queries](/es/docs/core-features/saved-queries) | Conservar SQL que debe reutilizarse. |

## Operaciones ClickHouse

- [Monitoring](/es/docs/core-features/monitoring): actividad, latencia, consultas lentas y errores.
- [Privileges](/es/docs/core-features/privileges): usuarios, roles, permisos y grants de clúster.

## AI Chat frente a agentes externos

AI Chat trabaja dentro de la UI con la conexión, pestaña, Schema y resultado actuales. Los agentes externos usan MCP o Dory CLI, crean Agent Runs y conservan SQL y resultados para la entrega humana. Consulta [Agent Workspaces](/es/docs/agents).
