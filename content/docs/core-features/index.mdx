# SQL Workspace

Dory remains a fast, familiar SQL client for people. The same workspace also receives SQL and results created by agents, so you can review and continue automated work without reconstructing it from chat.

## Daily SQL Surfaces

| Feature | Use it for |
| --- | --- |
| [Connections](/en/docs/core-features/connections) | Manage database credentials, network settings, and local files. |
| [Explorer](/en/docs/core-features/explorer) | Inspect databases, schemas, tables, columns, previews, and object details. |
| [SQL Console](/en/docs/core-features/sql-console) | Write, run, debug, and save SQL in multiple tabs. |
| [Charts & Results](/en/docs/core-features/charts-results) | Inspect real rows, filter results, and build charts. |
| [AI Chat](/en/docs/core-features/ai-chat) | Generate, explain, fix, or reshape SQL inside Dory. |
| [Saved Queries](/en/docs/core-features/saved-queries) | Preserve SQL that should become a reusable asset. |

## ClickHouse Operations

- [Monitoring](/en/docs/core-features/monitoring): inspect activity, latency, slow queries, and errors.
- [Privileges](/en/docs/core-features/privileges): manage ClickHouse users, roles, permissions, and cluster grants.

## AI Chat vs. External Agents

AI Chat works inside the current Dory UI with the selected connection, tab, schema, and result context. External agents connect through MCP or Dory CLI, create Agent Runs, and preserve SQL and results for human handoff. See [Agent Workspaces](/en/docs/agents) for the external-agent path.
