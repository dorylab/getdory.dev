# AI Chat

AI Chat 是 Dory 的智能分析助手，不只是一个普通聊天框。它能够结合当前数据库连接、Schema 结构、正在编写的 SQL，以及查询结果上下文，帮助你更快完成从问题到答案的全过程。

## 为什么使用 Dory AI Chat

传统 AI 工具通常不了解你的真实数据库结构，需要你手动提供表名、字段和上下文。Dory AI Chat 可以直接结合当前工作区信息，生成更准确、更可执行的回答。

你可以获得：

- 更准确的 SQL 生成结果。
- 对当前查询的实时解释与优化建议。
- 更自然的数据分析对话体验。
- 基于结果继续追问的连续分析能力。
- 减少手动描述 Schema 的成本。

## AI Chat 理解哪些上下文

当数据库已连接时，AI Chat 可以结合以下信息：

- 当前数据库与 Schema
- 表结构与字段信息
- 当前 SQL Console 中的查询语句
- 查询执行结果（如已运行）
- 当前分析目标与历史对话内容

这让回答更贴近真实业务场景，而不是通用模板答案。

## 核心能力

### 1. Generate SQL

直接使用自然语言生成查询，例如：

- 统计最近 7 天订单趋势
- 找出活跃用户 Top 20
- 按城市统计收入并按月分组
- 生成适合折线图展示的 SQL

### 2. Explain SQL

理解复杂 SQL 时，你可以让 AI：

- 解释每个子查询作用
- 说明 JOIN 逻辑
- 分析窗口函数用途
- 输出更易读版本

### 3. Fix SQL Errors

当 SQL 报错时，可直接让 AI 协助排查：

- 语法错误
- 字段不存在
- 表名错误
- 类型不匹配
- 聚合逻辑问题

### 4. Optimize SQL

针对慢查询或复杂查询，AI 可提供：

- 重写查询结构
- 减少扫描数据量
- 优化 JOIN 顺序
- 推荐索引思路（适用于支持索引数据库）
- 提升可维护性

### 5. Continue Analysis

查询完成后，你可以继续追问：

- 为什么这个指标下降了？
- 再按渠道拆分看看。
- 按周维度重新统计。
- 找异常值最高的地区。

这让分析过程更加连续，而不是每次重新开始。

## 推荐使用方式

### 与 SQL Console 配合使用（推荐）

1. 在 SQL Console 编写或运行查询。
2. 打开 AI Chat。
3. 让 AI 解释、优化或继续分析。
4. 将生成 SQL 回到编辑器执行。

### 独立提问

直接描述业务问题，让 AI 帮你生成第一版查询。

### 使用表上下文

在连接工作区中，AI Chat 可以结合当前数据库和表结构回答问题。你可以先在 Explorer 中确认目标表，再让 AI 生成 SQL 或解释字段含义。

## 高质量提示词示例

- 用 ClickHouse 统计最近 30 天 DAU 趋势。
- 解释这段 SQL 为什么重复计数。
- 优化这条 PostgreSQL 查询性能。
- 基于当前结果生成柱状图 SQL。
- 把这个复杂查询拆成多个 CTE。

## 最佳实践

- 连接真实数据库后再使用，效果最佳。
- 明确说明时间范围、指标口径与维度。
- 对生成 SQL 先审核再执行。
- 把 AI 当分析搭档，而不是黑盒自动化工具。
- 多轮追问通常比一次性长提示词更有效。
- 生产环境中优先让 AI 生成只读查询，并保留 `LIMIT` 或明确时间范围。
