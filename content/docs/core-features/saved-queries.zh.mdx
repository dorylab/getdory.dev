# Saved Queries

Saved Queries 能把一次性的分析沉淀为可复用资产。它适合需要反复执行、分享给团队，或继续扩展为后续分析的查询。

## 可以保存什么

- SQL 文本。
- 查询标题和描述。
- 所属连接和上下文。
- 文件夹分组。
- 标签和排序位置。

## 什么时候应该保存

- 这条查询回答的是重复出现的业务问题。
- 这条查询花了不少时间调优。
- 团队里其他人也会需要它。
- 这条查询会继续用于图表或报表流程。

## 命名建议

尽量用业务结果来命名，而不是只写表名。

好的命名示例：

- 最近 30 天 DAU 趋势
- 支付失败订单排查
- ClickHouse 慢查询 Top 50
- 每日新增用户与转化率

## 使用流程

1. 在 SQL Console 中完成并验证查询。
2. 点击保存操作。
3. 填写清晰标题和必要描述。
4. 选择或创建文件夹。
5. 后续从 Saved Queries 侧边栏打开、运行或继续编辑。

## 团队实践

- 将生产排查 SQL 和日常指标 SQL 分开管理。
- 对有副作用的 SQL 添加明显说明；生产环境优先保存只读查询。
- 对依赖特定时间范围或业务口径的 SQL，在描述中写清楚假设。
- 定期归档不再使用的旧查询，减少搜索噪音。
