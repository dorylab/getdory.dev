# 审阅并继续 Agent 工作

Agent 通过 Dory MCP 探索 Schema 或运行 SQL 后，使用这个工作流完成人类交接。

## 1. 打开 Agent Run

使用 Agent 返回的 workspace URL 或 work ID。确认标题与请求一致，状态为 completed 或清楚说明了阻塞原因。

## 2. 阅读 Findings 与 Steps

先查看选中的连接、已检查表、执行 SQL、时间范围、过滤条件、采样、截断和未解决假设，再打开具体标签页。

## 3. 检查准确 SQL

打开相关 SQL 标签页，与 Explorer 中的表名、字段和 Join 对照，确认数据库正确、查询只读且范围合理。

## 4. 验证结果

检查真实返回行、字段类型、空值、重复值、总量和异常分类。表述流畅的 finding 不能替代证据验证。

## 5. 编辑并重新运行

假设或过滤条件需要修正时，直接在 Dory 工作台编辑 SQL。先重跑小范围版本，结果结构正确后再扩大范围。

## 6. 继续工作

- 结果适合可视化审阅时添加筛选或图表。
- 将稳定、可复用的 SQL 保存为 Saved Query。
- 一次性调查细节留在 Agent Run 中。
- 后续问题已经变成不同任务时创建新的 Agent Run。

当人类能解释查询、信任证据、复现结果并直接继续工作时，交接完成。
