# エージェントの作業をレビューして続ける

エージェントが Dory MCP で Schema を探索または SQL を実行した後に使います。

## 1. Agent Run を開く

返された workspace URL または work ID から開きます。タイトルが依頼と一致し、status が completed または明確な blocked であることを確認します。

## 2. Findings と Steps を読む

接続、テーブル、SQL、期間、filter、sample、切り詰め、未解決の前提を確認します。

## 3. 正確な SQL を確認する

関連タブを開き、Explorer の名前や join と比較します。DB の選択、読み取り専用、範囲が適切か確認します。

## 4. 結果を検証する

実際の行、列型、null、重複、合計、予期しないカテゴリを確認します。findings の文章だけでなく証拠を検証してください。

## 5. 編集して再実行する

前提や filter が違う場合は Dory で直接 SQL を修正します。小さい範囲で再実行し、正しい結果形状を確認してから拡大します。

## 6. 続ける

- 必要に応じて filter や chart を追加します。
- 再利用できる SQL は Saved Query にします。
- 一度限りの調査は Agent Run に残します。
- 別の問いになったら新しい Agent Run を作成します。

人間がクエリを説明し、証拠を信頼し、再現して続けられれば引き継ぎ完了です。
