# エージェントでデータベースを分析する

Schema を推測せず、SQL と証拠をチャットだけに残さずにデータ質問へ答えるワークフローです。

## 1. 問いを限定する

ビジネス上の問い、期間、データ機密性、読み取り専用の条件を明示します。「すべて分析する」から始めないでください。

## 2. Agent Run を作成する

エージェントは問いを表すタイトルで `dory_create_work` を呼び、返された `workId` を後続のすべての処理に使います。

## 3. 接続を選ぶ

検証済み `connectionId` がなければ `dory_list_connections` を使い、接続名と DB 種別を確認します。

## 4. Schema を先に探索する

`dory_explore_schema` でテーブル、列、型、join key、日付列、規模を確認します。必要最小限の行だけ preview します。

## 5. 安全な SQL を実行する

`dory_run_readonly_sql` で明示的な列、狭い期間、適切な集計、行数制限を使います。大きなログ テーブルは sample または短期間から始めます。

## 6. 証拠を保存する

意思決定に必要な SQL と結果をわかりやすいタブへ保存します。一度限りの探索は Agent Run に残し、再利用する SQL だけを Saved Query にします。

## 7. 完了してレビューする

`dory_finish_work` に findings、steps、前提、sample や切り詰めを記録します。Dory で Agent Run を開き、意思決定の前に SQL を確認してください。

人間がチャットを再構築せずに、正確な SQL と結果から続けられれば完了です。
