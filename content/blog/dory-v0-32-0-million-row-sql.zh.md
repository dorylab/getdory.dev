# Dory v0.32.0：百万行 SQL 结果，也能流畅探索

![Dory v0.32.0 通过 DuckDB 风格的计算核心，将大型 SQL 结果集转化为基于 Parquet、可流畅浏览的数据](/images/blog/dory-v0-32-0-no-limit-sql-cover-zh.png)

Dory v0.32.0 现已发布。

这一次，我们解决了一个简单却长期困扰数据工作者的问题：**当 SQL 查询返回的数据很多时，为什么一定要在“完整结果”和“流畅体验”之间做选择？**

在 v0.32.0 中，你可以运行 **No-limit SQL queries**，获得完整查询结果；即使面对百万行数据，也可以直接在 Dory 中流畅浏览、筛选、排序、生成图表和导出。

不再为了避免页面卡顿而提前截断数据，也不必把查询结果搬到另一个工具才能继续分析。

## 不只看前 200 行

为查询设置一个默认行数上限，是避免误查大表的有效保护。但当你明确需要完整结果时，这个限制也会成为阻碍。

现在，你可以在 Dory 的查询设置中选择 **No limit**。Dory 将不再自动为 SQL 添加 `LIMIT`，让查询按照你的原始意图执行。

无论是导出完整订单明细、排查分散在大表中的异常记录，还是为后续分析准备全量数据，你都不需要再手动绕过工具的结果限制。

日常探索时，你仍然可以保留默认行数限制；真正需要完整数据时，再一键切换到 No limit。速度与完整性，由你决定。

## 百万行结果，依然可以顺畅浏览

查询出一百万行并不难，难的是让这一百万行真正可用。

传统结果表格常常会尝试把大量数据一次性加载到浏览器。结果越大，等待越久，内存占用越高，滚动、筛选和排序也会越来越迟钝。

Dory v0.32.0 改变了结果表格的加载方式。打开一个大型 ResultSet 时，Dory 只读取并展示你当前正在查看的部分，并在你滚动时提前准备接下来的数据。

对你来说，它仍然是一张完整的表格：

- 快速滚动和定位百万行结果
- 对完整结果进行筛选、搜索和排序
- 基于完整数据生成图表，而不只是前几百行
- 随时导出筛选后的 CSV 或 Parquet 文件

数据量变大，操作方式不需要改变。你看到的是完整结果，浏览器承担的却始终只是当前需要展示的一小部分。

## 查询结束，结果不会消失

过去，SQL 结果往往只是一次临时响应。关闭标签页、刷新页面，或者切换工作区之后，这份结果可能就需要重新查询。

在 v0.32.0 中，Dory 会将完整 ResultSet 保存为 Parquet。查询完成后，结果仍然保留在你的工作区中，可以重新打开并继续使用。

这意味着你可以：

- 今天运行查询，明天继续分析
- 在不同 SQL 标签页之间切换，而不丢失已有结果
- 保留筛选、排序和选择状态
- 直接基于已有结果继续制作图表或导出文件

一次查询，不再只是一次性的输出，而是可以反复探索和复用的数据资产。

## DuckDB 正在成为 Dory 应用架构的核心

为了实现这套体验，我们在 v0.32.0 中将 DuckDB 引入了 Dory 应用架构的核心。

它不只是一个用来加速查询的幕后组件，也不只是 Dory 新增支持的一种数据库。DuckDB 正在成为 Dory 连接数据库结果、文件与后续分析能力的统一计算基础。

目前，Dory 使用 Parquet 存储完整 ResultSet，并由 DuckDB 负责这些大型结果集的读取和计算。

当你筛选、排序或搜索百万行结果时，Dory 不会先把所有数据发送到浏览器。DuckDB 会直接在 Parquet 结果集上完成操作，再将当前需要的数据返回给界面。

这也是为什么大型 ResultSet 不仅能被保存，还能继续快速地浏览、分析和导出。

你不需要配置 DuckDB，也不需要手动管理 Parquet 文件。它们就在 Dory 背后工作，让完整结果和流畅体验可以同时存在。

更重要的是，这为 Dory 接下来的文件分析能力打下了基础。DuckDB 天然适合直接分析 Parquet、CSV 等文件；未来，我们将基于这套架构继续加强对本地文件和多种文件格式的支持，让用户无需先导入数据库，也能直接打开文件、运行 SQL、组合数据并完成分析。

v0.32.0 的百万行 ResultSet 是这套新架构带来的第一批能力。未来，数据库查询结果和文件数据将在 Dory 中拥有更一致的分析体验。

## 从查询到分析，不再被数据量打断

Dory v0.32.0 将大结果集体验连接成了一条完整链路：

1. 使用 **No limit** 运行完整 SQL 查询。
2. 在查询过程中持续看到结果预览和执行进度。
3. 完整 ResultSet 自动以 Parquet 保存。
4. 在 Dory 中流畅浏览百万行数据。
5. 继续筛选、排序、制作图表或导出。

从发起查询到获得洞察，你不再需要因为结果太大而中断工作，也不需要在多个工具之间来回搬运数据。

与此同时，以 DuckDB 为核心的应用架构，也让这条链路不再局限于数据库查询。接下来，我们会把同样流畅、直接的分析体验带到更多文件数据中。

## Dory v0.32.0 现已发布

数据不应该因为太多，就变得无法使用。

Dory v0.32.0 让你既能保留完整结果，也能拥有快速、顺畅的分析体验；它也标志着 DuckDB 正式进入 Dory 应用架构的核心，为下一阶段的文件分析能力做好准备。
